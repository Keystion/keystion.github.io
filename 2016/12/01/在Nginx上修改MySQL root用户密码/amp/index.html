在Nginx上修改MySQL root用户密码

2016-12-01

在/etc/my.cnf文件的[mysqld]下面（换行）添加skip-grant-tables，保存并退出（ESC->:->wq->回车）。

之前

之后

重启MySQL：

成功提示：

Stopping MySQL: [ OK ] Starting MySQL: [ OK ]

接下来继续输入：mysql；输入

完成

重启MySQL

输入mysql -u root -p就会提示输入新密码，输入刚才设置的秘密就可以进去，说明设置成功。

此时你以为结束，并没有，记得把skip-grant-tables从/etc/my.cnf里面删除掉，不然即使你通过刚刚的过程修改了root用户的密码最后还是无法对数据库进行增、删、改。
